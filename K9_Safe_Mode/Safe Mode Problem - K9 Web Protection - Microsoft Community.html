    Safe Mode Problem - K9 Web Protection

    Hello,

    I have downloaded a program called K9 Web Protection and it works fine in normal mode but in Safe Mode it doesn't load at all. I did not have this problem with Windows 7 but only now as I am using Windows 8.1. Is there any way I can ensure K9 Web Protection loads in Safe Mode or, better still, can I make it so I need an administrator password to enter safe mode at all? I recall previously only being able to access Safe Mode on an already administrator account; however, this no longer is the case. 

    Thanks.

    Hello Matt, 

    In Safe Mode, only specific programs and files needed to run the operating system are loaded. Startup programs don't run in safe mode, and only the basic drivers needed to start Windows are installed.

    K9 Web Protection is an Internet filtering software it may need internet to work. As a work around I would suggest you to boot into safe mode with networking and check if it works there.

    To go into safe mode with networking refer to the link given below

    Windows Startup Settings (including safe mode)

    http://windows.microsoft.com/en-us/windows-8/windows-startup-settings-safe-mode

    Hope this information is helpful. Please do let us know if you need further assistance with Windows, we’ll be glad to assist you.

    Thank you.

    Hi Matt,

    You can change which drivers load in safe mode. See this page (Be sure to back up the registry before and after doing this):

    http://www.discuss.org/forum/discuss-asp/k9-web-protection-administration-safe-mode-net-working-1338641.html

    Independently, I did it the long way:

    Open regedit.

    Note the registry values in:


    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\bckd

    and

    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\bckwfs,

    and add them to:


    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\SafeBoot\Network

    I'm sure it's quicker to do this by following the instructions in the link above. I added all the registry values from bckd and bckwfs - instead of just the image path. And I had to repair k9 twice (installing over the old installation) before k9 worked properly again, i.e, that it blocked everything it's supposed to block. But now it works in safe mode and in (normal) mode.
